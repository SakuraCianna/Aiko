# 待开发

当前 Aiko 已进入 Agent 能力加固期。主链路已经包含 LangChain Agent Runtime, LangGraph 审批流, Planner / Retriever / Executor, ResearchAgent / MemoryAgent, 长期记忆, Tavily MCP, Open-Meteo 天气, 腾讯云 ASR/TTS, VRM 渲染, 动作确认, SQLite 审计日志和动作审计面板。

下面只保留基于当前代码状态还需要继续推进的事项。

## 1. 实时语音体验继续产品化

已完成:

- 腾讯云 ASR provider 已接入
- 麦克风录音停止后会自动提交 WAV 附件进入 ASR 和 Agent 链路
- 腾讯云 TTS provider 已接入
- 默认音色为超自然大模型音色 `603007 邻家女孩`
- TTS provider 不可用时会回退到浏览器 Web Speech API

未完成:

- 还不是完整低延迟双工实时语音
- 仍以录音片段提交为主
- 没有流式 ASR
- 没有播放队列
- 没有分句 TTS 缓存
- 没有语音打断和口型同步闭环
- 当前录音实现还会触发 `ScriptProcessorNode` deprecated 提示

下一步:

- 把录音实现升级到 `AudioWorkletNode`
- 增加播放队列和中止播放
- 增加分句 TTS 缓存
- 把 TTS 播放进度同步到 VRM 口型

## 2. 高风险 Windows 能力继续收紧边界

已完成:

- 文件读取, 文件写入, 删除到 Aiko trash, 从 Aiko trash 恢复, 目录列举和受控 PowerShell 命令都已经进入本地执行器和权限矩阵
- Planner 可以确定性识别明确的目录列举, 文件读取, 文件写入和 PowerShell 命令请求
- 高风险动作不会被永久授权
- 高风险动作会进入确认弹窗和动作审计面板
- 文件覆盖写入前会自动备份旧内容
- 删除文件会写入 Aiko trash 恢复元数据
- 从 trash 恢复会检查目标路径是否已经存在
- Shell 命令已收紧到只读 allowlist
- Shell 命令会拒绝非 allowlist 命令, 管道, 重定向, 分号, 嵌套 shell 和敏感路径目标
- 动作审计面板支持按风险, 能力, 结果和关键词筛选
- 动作审计面板会展示备份路径, trash 隔离路径和按 exit code, stdout, stderr 分级的 Shell 输出
- 动作审计面板会展示 Aiko trash 恢复历史, 区分待恢复和已恢复记录
- 动作审计面板可从待恢复记录中准备恢复动作, 仍然走统一高风险确认弹窗
- 高风险确认弹窗会按能力展示本次动作的具体影响范围和恢复策略

未完成:

- Shell 命令还没有完整撤销后台
- 高风险确认文案后续还可以继续扩展成可视化影响预览
- 窗口控制, 截图分析和键鼠自动化仍未开放

下一步:

- 给 Shell 命令补完整撤销或补救建议后台
- 给恢复历史增加按文件名和状态筛选
- 给高风险动作增加可视化影响预览
- 在开放窗口控制和键鼠自动化前, 先设计独立的 critical 风险层

## 3. Agent 自主任务流继续增强

已完成:

- 多步骤低风险动作可以拆成批量待确认动作
- 长回答会自动转成桌面 `Aiko` 文件夹里的带时间戳 Markdown 写入动作
- 长期记忆已接入 SQLite 和 `sqlite-vec` 兼容向量索引
- 体验策略会根据用户语气进行短期调整
- Agent trace, run lifecycle, action journal 和 worker 摘要已经可在调试面板查看

未完成:

- 还没有完整任务运行面板
- 没有子任务失败重试
- 没有任务暂停和恢复
- 没有用户可编辑执行计划
- 内部 worker 子 Agent 和 hook 机制已有雏形, 但 Tavily, 文件写入, 长规划和多步骤操作还没有全部纳入统一 worker 编排

下一步:

- 做任务运行面板
- 给 worker 增加统一调度和重试策略
- 给复杂任务增加计划确认和执行记录
- 让用户能在执行前编辑计划

下一步:

- 给 idle, listening, thinking, speaking, executing, success, error, dragging 分别定义更细动作
- 把 TTS 播放状态映射到口型和表情

# 待开发

当前项目已经进入 MVP 增强期, Agent 主链路, 本地动作确认, 长期记忆, VRM 渲染, Tavily MCP 搜索和 Open-Meteo 天气工具都已经接入。下面三项是下一阶段优先级最高的未完成内容。

## 1. 专业实时语音理解

- 目标: 让用户点击麦克风后可以稳定进行实时语音交流, 而不是只依赖浏览器 Web Speech API 或录音附件占位。
- 现状: 渲染层已有 Web Speech API 入口和录音附件兜底, 但主进程还没有接入真正的 ASR provider。
- 下一步: 确定 ASR 服务或本地模型, 设计流式转写接口, 把转写结果接入现有 `ChatPayload` 和 Agent 上下文。

## 2. 专业 TTS 和 zero-shot voice cloning 回复链路

- 目标: 用高实时性的专业 TTS 替代文字气泡主展示, 后续支持 zero-shot voice cloning。
- 现状: 当前只有基础 `speechSynthesis` 语音输出, 还没有进入完整的流式回复播放链路。
- 下一步: 设计文本分段, 播放队列, 中止播放, 口型同步和缓存策略, 再接入选定的 TTS/voice cloning provider。

## 3. 更完整的系统自动化和权限策略

- 目标: 让 Aiko 能在用户确认和授权规则保护下执行更多 Windows 本地任务, 但不变成失控的大型 Agent。
- 现状: 已支持打开应用, 打开网页, 默认应用选择, 提醒, 写入桌面 Markdown 等低风险能力。
- 已推进: Agent Trace 和 Action Journal 已接入 SQLite 审计仓库, 调试面板后续可以基于持久化审计数据继续增强。
- 下一步: 扩展应用发现, 文件操作, 窗口操作和多步骤任务执行, 同时完善权限分级, 默认选项管理和提示词注入防护。

# 待开发

这个文件只记录当前代码基线之外的后续增强项。上一轮待开发主线已经基本收口: Agent 生命周期, 操作日志, 权限矩阵, Active Memory, Commitments, Heartbeat, 内部 worker, hook, 高风险 Windows 能力, 语音入口, VRM 动作联动和用户可见任务卡片都已经进入代码。

## 已完成的当前基线

### Agent 稳定性

- LangChain + LangGraph Runtime 已接入, Aiko 对外保持单角色, 内部拆成 Retriever, Planner, Executor, ResearchAgent, MemoryAgent 和 worker registry.
- Run lifecycle 已记录 accepted, running, waiting_approval, completed, failed, cancelled.
- Action journal 已记录 planned, approval, execution.
- Runtime hooks 已覆盖 model call, tool planning, memory write.
- Worker registry 已支持运行记录, 最近记录裁剪和失败重试.
- Research, multi-step, desktop markdown, file operation, memory write, experience reflection, commitment 都已经纳入 worker 调度.
- 确定性 Planner 已支持一句话拆多个低风险或需确认动作, 例如打开应用, 打开网页, 创建提醒, 截屏查看桌面.
- 多步骤动作会包装成 batch_actions, 批量风险继承子动作最高风险, 包含 critical 时不会降级.
- 用户可以在确认弹窗里取消批量计划里的部分步骤, 只能删除原步骤, 不能改写, 重排或新增步骤.

### 高风险 Windows 能力

- 已支持打开应用, 打开 URL, 创建提醒, 取消提醒.
- 已支持文件读取, 文件写入, 目录列举, 删除到 Aiko trash, 从 Aiko trash 恢复.
- 已支持受控 PowerShell 命令, 只允许单条只读 allowlist 命令.
- 已支持 critical 风险层: 截屏, 窗口控制, 键盘输入, 鼠标输入.
- 高风险和 critical 动作默认不能永久授权.
- 确认弹窗会展示影响预览.
- 审计面板可以筛选风险, 能力, 结果, 关键词, 并查看备份和 trash 恢复信息.

### 记忆与陪伴

- 长期记忆已使用 Node 24 `node:sqlite`.
- 向量检索已通过 `sqlite-vec` 优先启用, 不可用时降级到 JSON 稀疏向量.
- Active Memory 会从长期记忆里选择当前轮更相关的内容.
- MemoryAgent 负责记忆召回和候选记忆写入.
- ExperiencePolicy 会根据用户语气推断满意度倾向, 用于短期回复策略调整.
- CommitmentService 会从对话中捕获软承诺.
- Heartbeat 主动陪伴已接入, 默认 24 小时最多一次, 支持安静时段和是否朗读.

### 语音

- 麦克风录音已使用 AudioWorklet, 不再使用废弃的 ScriptProcessorNode.
- Renderer 已支持边录边切 PCM16 分片, 推送给主进程 ASR streaming session.
- 当前腾讯云 ASR 实现仍是 buffered 兼容层: 接收分片, 结束后合成 WAV 并调用一句话识别.
- 腾讯云 TTS 已接入, 回复可分句排队播放.
- TTS 不可用时会降级到浏览器 Web Speech.
- TTS 播放会驱动 VRM 口型.
- 中止回复会停止当前语音播放.
- TTS 已有基础缓存 provider, 避免重复短句反复生成.

### 桌宠表现

- VRM + Three.js + `@pixiv/three-vrm` 已替代 Live2D 路线.
- `assets/vrm/Aiko.vrm` 作为默认模型入口, 模型文件已被 gitignore 忽略.
- 已支持 lookAt, idle, listening, thinking, speaking, dragging, confirming, waiting, searching, writing, success, failure, recovering 等行为.
- 已支持 greet, nod, think, search, write, explain, present, proud, interrupt, errorRecover, idleShift, wait, focus 等一次性动作.
- Agent 阶段事件会驱动角色动作.
- 等待用户确认时会切换到 waiting/wait.
- 空闲时会低频播放待机小动作.
- 主动陪伴消息会触发 notice 或 curious 动作.

## 后续增强项

下面是下一阶段增强, 不是当前基线阻塞项。

### 1. 语音实时性继续增强

- 接入腾讯云实时 ASR WebSocket provider, 获取真实 partial transcript.
- 把 partial transcript 显示到输入区或任务卡片.
- 做更自然的打断, 例如用户开口时自动降低或停止 TTS.
- 把口型从时间模拟升级为音量或音素驱动.
- 将 TTS 缓存持久化到磁盘, 并增加缓存清理策略.

### 2. 高风险能力产品化继续增强

- 为 Shell 命令建立更完整的撤销建议后台.
- 为窗口控制, 键盘输入, 鼠标输入增加更细的确认文案和二次保险.
- 截屏后接入多模态模型分析链路, 当前只负责安全截图和保存.
- 为 critical 操作增加更明显的任务卡片展示和失败恢复建议.

### 3. Agent 评估与可观测性

- 增加一组端到端场景测试, 覆盖一句话多步骤, 权限拒绝, 中止, fallback 模型, Tavily, Open-Meteo 和长文写入.
- 增加提示词注入红队用例, 覆盖网页搜索结果, 附件, 长期记忆和文件内容注入.
- 把 debug 面板进一步拆成用户视角任务面板和工程视角 trace 面板.

### 4. 桌宠存在感继续打磨

- 为等待, 被拖拽, 长时间空闲, 任务失败恢复增加更细动作组合.
- 根据 Aiko 的情绪和语气选择 TTS 语速, 音高和动作.
- 给任务卡片增加更自然的进度文案, 比如正在查资料, 正在准备安全动作, 等你点头.
